# Blog 編集ガイドライン

## 筆者の文体

このブログは個人ブログであり、筆者の声・温度感を最優先する。

### 守るべき文体の特徴
- 基本は「だ/である」ベースにカジュアルな語尾（〜なーって感じ、〜ってのはある、〜だよね、〜かなー）
- 思考の流れをそのまま書く口語的なスタイル
- 自虐や正直な感想をそのまま出す（「飽きた」「使ってない」「わからない」など）
- 見出しも堅くしない（「とかまぁいってみたが」のようなノリはそのまま残す）

### 触っていいこと
- 1文が長すぎる場合は区切る（目安: 80文字超えたら分割を検討）
- 1段落に複数の話題が混在してる場合は分ける
- 誤字脱字の修正（ぷらん → プラン）
- 固有名詞の表記統一（claude → Claude、suno → Suno、iphone → iPhone）
- です/ます と だ/である の混在を統一（だ/である側に寄せる）
- 不規則な空行（2連続の空行など）の整理
- 冗長な繰り返し表現の軽い整理（「〜なーって感じはあった」が連続する場合など）

### 触ってはいけないこと
- 語尾のカジュアルさ（〜ってのはある → 〜という印象 のような書き換えはしない）
- 見出しのノリや温度感
- 筆者の意見・感想の内容そのもの
- 文体を「読みやすい記事」風に均質化すること

## メタデータ

### ファイル名
`{番号}_{slug}.md` の形式。slug は英語ケバブケースで内容を表すもの。

### frontmatter

```yaml
---
pubDatetime: 2026-01-30T15:10:00+09:00  # ISO 8601 + タイムゾーン
title: '記事タイトル'
featured: true  # トップに出すかどうか
slug: kebab-case-slug  # URL用。ファイル名のslug部分と一致させる
draft: false
tags:
  - タグ1
  - タグ2
description: 1〜2文で記事の内容を要約。筆者の声が出る書き方でOK。
---
```

### プレースホルダーは残さない
slug, tags, description にはすべて記事の内容に合った値を入れる。`slug: slug` や `tags: - tag` や `description: 記事の説明` のようなプレースホルダーは必ず埋める。

### タグの付け方
既存タグから選ぶのを優先する。よく使われているタグ:
- 雑記、内省、開発、個人開発、AI、インフラ、体験談、ポエム
- 技術固有: k8s, React Native, playwright, nextjs など

### description
記事の内容を1〜2文で。堅くなりすぎなくていい。「〜した話」「〜してみた感想」のようなカジュアルさでOK。
